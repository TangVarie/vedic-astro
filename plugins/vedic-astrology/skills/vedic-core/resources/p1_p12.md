## P1 角色身份 [关键权重]

行星的"职务"由它管理的宫位决定：

```
Core-Driver (1宫主) = 忠诚者
  → 永远服务盘主，力量越强越好

Yogakaraka (同掌吉凶宫) = CEO
  → 同时管三角宫和角宫，最高价值

Faithful (5/9宫主) = 盟友
  → 三角宫主，天然吉利

Trader (2/4/7/10宫主) = 交易者
  → Kendradhipati法则：吉星管角宫→变怠工，凶星管角宫→变自律

Growth-Hacker (3/6/11宫主) = 竞争者
  → 带毒增长，力量越强副作用越大

Trika-Lord (8/12宫主) = 清理者 / 转化者
  → 擅长结束、清理、转化、科研/审计/海外有价值
  → 双标签设计：内部分析时用学术中性词 "Trika-Lord"
                面向用户输出时用 "清理者" 或 "转化者"，避免 "Destroyer" 的情绪色彩
  → 历史背景：v2 之前用 "Destroyer" 单标签，该词在英文里自带负面暗示，
              导致 LLM 容易把 8/12 宫主一律往灾难方向解。v3 改双标签后，
              内部代码保持精确（Trika 是梵文术语，指 6/8/12 三凶宫，此处特指 8/12），
              输出用 "清理者/转化者" 突出其建设性职能。
```

**复合身份（面向用户的翻译）**：
- Soft-Betrayal = "温和错位"（吉星担凶职）→ 好心但用错了地方
- Hard-Construction = "高压建设者"（凶星担吉职）→ 苦劳出成果
- Growth-Hacker = "成长型竞争者" → 越用越强但有内耗

### P1.3 纹理审计（必须标注，不可省略）

检测自然属性与职能的冲突，这决定了执行过程的"体感"：

```
Soft-Betrayal（欺骗性风险）：
  触发条件：自然吉星（Jupiter/Venus/Mercury/Moon）担任 GH 或 Trika-Lord 角色
  典型案例：
    Jupiter掌6 = 过度扩张导致崩盘
    Venus掌8 = 沉溺感官导致毁灭
    Jupiter掌12 = 无节制的精神追求导致现实脱节
  审计要求：分析中必须标注"欺骗性风险"
  人话翻译："这颗星看着友善，但它管的事不友善"
  禁止：美化为"转化的力量""深层的智慧"

Hard-Construction（高压红利）：
  触发条件：自然凶星（Saturn/Mars/Rahu/Ketu）担任Core-Driver或Yogakaraka角色
  典型案例：
    Saturn为Yogakaraka = 极度自律出成果
    Mars为Core-Driver = 过程痛苦但结果是真的
  审计要求：分析中必须标注"高压红利"
  人话翻译："过程苦但结果是真的"
  禁止：淡化过程的痛苦，不能只说结果好
```

### P1.4 Maraka警示（健康分析时必须挂载）

```
触发条件：行星掌管2宫或7宫

规则：
  健康/寿命/重大意外分析时 → 挂载[Maraka]标签
    含义：该星在健康相关的Dasha期间有"终止"信号
    人话："这颗星管的领域跟你的身体安全有关联"

  事业/财富/家庭分析时 → 维持Trader身份
    不挂载Maraka标签
    人话："赚钱和谈合作的时候它就是个正常的执行者"

语境切换硬约束：
  同一颗星在不同分析板块中身份不同
  例：Venus管2宫+7宫
    健康板块 → [Maraka]"需关注Venus大运期间的健康波动"
    财富板块 → [Trader]"稳定的收入管理者"
    婚姻板块 → [Trader+7宫]"伴侣的品质由Venus状态决定"
  禁止在事业分析中提到"杀手""终止"等健康语境的标签
```

### P1.5 Functional Benefic / Malefic 矩阵（v3 新增，必须执行）

P1 的 6 个角色身份（Core-Driver / Yogakaraka / Faithful / Trader / Growth-Hacker / Trika-Lord）
是基于"管哪个宫"的吉凶简化。但 KN Rao 体系强调的另一维度是 **Functional 性质**——
即"对于这个特定的 Lagna，每颗行星实际上是吉是凶"。

这两个维度独立运作：
  - P1 角色 = "这颗星在公司里担任什么职务"
  - Functional 性质 = "这颗星本质上是公司的资产还是负债"
  
一颗星可以同时是 Growth-Hacker（角色带毒）但是 Functional Benefic（本质有利），
也可以是 Trader（角色中性）但是 Functional Malefic（本质拖累）。
PAC 联合判定时必须把这一维度纳入。

⚠️ 关键阅读规则（v3.3 强化）：

```
1. Mixed / School-disputed 列里的行星不能按 FB 或 FM 单边定调
   - 评估其行星状态时必须分两线：
     主导面（吉的宫主导）+ 污染面（凶的宫拖累）
   - 例：Cancer Jupiter (L6+L9) → L9 主导（智慧/dharma 福报）+ L6 污染（健康/敌对负担）
   - 例：Gemini Saturn (L8+L9) → KN Rao 偏 FB（L9 主导），但部分流派归 mixed

2. School-disputed 项的处理：
   - 默认按本系统底色 KN Rao 派
   - 但在最终输出"决断性结论"时，必须显式承认存在派系分歧
   - 例：Aries Moon (L4) → 本系统按 Neutral 处理，部分派归 FB（L4 Kendra lord）

3. Rahu/Ketu 不在矩阵：
   - 评估 Rahu/Ketu 时看落宫主（dispositor）+ 合相 + 主星行运
   - 不要给 Rahu/Ketu 强行套 FB/FM 标签

4. Lagna lord 特殊处理：
   - Lagna lord (L1 主) 永远是 Core-Driver，但 functional 性质看其管的第二宫位
   - 例：Aries Mars L1+L8 = Core-Driver + 8H contamination
   - 例：Cancer Moon L1 only = pure FB (单宫主)
   - 例：Libra Venus L1+L8 = Lagna lord but mixed

5. Yogakaraka 的严格定义：
   - 同时管一个 Kendra (1/4/7/10) + 一个 Trine (5/9) 的行星
   - ⚠️ Lagna lord 即使管了 Trine 也不算 Yogakaraka（因为它是 L1，已经是 Core-Driver）
   - 完整 Yogakaraka 共 6 个：
       Taurus Saturn (L9+L10)
       Cancer Mars (L5+L10)
       Leo Mars (L4+L9)
       Libra Saturn (L4+L5)
       Capricorn Venus (L5+L10)
       Aquarius Venus (L4+L9)
   - Pisces Mars (L2+L9) 是"强 dharma/maraka mixed benefic"，因不掌 Kendra，
     不归入 Yogakaraka
   - "near-Yogakaraka"、"部分 Yogakaraka" 这类模糊标签在 v3.4 起废弃，
     一个行星要么是 Yogakaraka 要么不是。
```

#### 使用规则

```
1. 在 P1 审计每颗行星时，必须同时标注两层：
   - P1 角色（基于管宫）：Trader/GH/Trika-Lord/...
   - Functional 性质（基于本 Lagna）：FB / FM / Yogakaraka / Neutral

2. 联合判定的 4 种象限：
   
   FB + 强 Shadbala + 吉位 → "核心变现引擎 / 高可用优势星"，可大力依赖
   FB + 弱状态        → "本来有福但没发挥"，需要特定 Dasha 才能激活
   FM + 强 Shadbala   → ⚠️ "强力的本盘对手"，越强副作用越大
                         （这是最容易被误判的——AI 看到 Shadbala 高就说好）
   FM + 弱状态        → "天然被削弱，反而是好事"

3. 与 P1.3 纹理审计的关系（叠加，不冲突）：
   - P1.3 纹理 = 自然属性 × 管宫
   - P1.5 Functional = 管宫 × 本 Lagna
   - 一颗星可以同时触发 Soft-Betrayal 和 Functional Malefic 或 FB-mixed
     例：Cancer Lagna 的 Jupiter 管 L6 + L9
        - P1 角色：L9 主 = Faithful，L6 主 = GH
        - P1.5 Functional：FB-mixed（dharma benefic with L6 contamination）
          → L9 主导面：法律/学术/远方/智慧的祝福
          → L6 污染面：与服务、敌对、健康的负担相关
          → 不能简单标 FB 也不能标 FM
        - 推断 Jupiter 状态时必须分两线评估，写"FB-mixed"
   
4. 凶宫主大运的特例：
   FM 的凶宫主大运 = "标准凶大运"（按 house_framework 负面条件判定）
   FB 的凶宫主大运 = "凶宫成本但有 FB 补偿"，措辞可留余地
     例：Cancer Lagna 的 Mars 管 L5+L10（FB+Yogakaraka）
        Mars 大运负面期 ≠ 完全凶，因为 Mars 本身是 FB
        ⚠️ 注意：Cancer 上升的 L1 主是 Moon（不是 Mars）。
           Mars 只管 L5 和 L10，不管 L1。
```

### P2.2 Avasthas 行星状态（v2 新增，做精细 Dasha 预测时启用）

```
Avasthas = "状态" / "阶段"，指行星在特定状态下的具体表现倾向。
古典 Parashari 收录多套 Avasthas，本系统采用 KN Rao 临床精选的两套：

【Baladi Avastha（五阶段年龄态）】
  与 P8 年龄状态部分重叠但更精细，按行星在星座内的度数判定：
  
  奇数星座 (Ar/Ge/Le/Li/Sg/Aq)：
    Bala (婴儿) = 0-6°       → 力量 25%（潜力未展，事件偏理想化）
    Kumara (少年) = 6-12°    → 力量 50%（成长中，事件偏试探）
    Yuva (青年) = 12-18°     → 力量 100%（最强，事件最具影响力）
    Vriddha (成熟) = 18-24°  → 力量 50%（经验丰富但创新弱）
    Mrita (老化) = 24-30°    → 力量 0%（力量耗尽，事件偏结束/收尾）
  
  偶数星座 (Ta/Cn/Vi/Sc/Cp/Pi)：
    顺序反过来：Mrita(0-6°) → Vriddha(6-12°) → Yuva(12-18°) → Kumara(18-24°) → Bala(24-30°)
  
  使用：
    - 当一颗星在其 Dasha 期间事件偏离预期时，查 Baladi
    - Mrita / Bala 状态的行星 → 事件常"差一口气"或"提前结束"
    - Yuva 状态的行星 → 事件"足量足质"展开

【Jagrat-Swapna-Sushupti（觉醒三态）】
  按行星在自身星座 vs 友/敌座的状态判定：
  
  Jagrat (觉醒) = 行星在自家或入旺  → 事件清晰、主动、有掌控感
  Swapna (做梦) = 行星在友座或中性  → 事件模糊、被动、半推半就
  Sushupti (沉睡) = 行星在敌座或落陷 → 事件失控、被外力推动、难自主
  
  使用：
    - 解释为什么有些 Dasha 期间"明明数据看着好但感觉很迷"
    - Sushupti 状态的吉星 → 美意被埋没
    - Jagrat 状态的凶星 → 凶力被放大但也更可控（"敌人摆明了在打你"比"暗箭难防"好对付）

【何时启用 Avasthas】
  - core Step 1 P1-P12 审计：可选启用（若希望深度更细，加在 P2 段落）
  - timing Mode B Dasha 交叉验证：强烈建议启用（细化吉凶判定）
  - rectifier Step 3a 事件匹配：可选启用（用 Mrita/Bala 解释"应发未发"案例）
  - Q&A 追问"为什么这段时间感觉不对"：必启用
  
  默认 core 主流程不必每颗星都跑 Avasthas，避免过载。
  当某颗星的事件预测与用户反馈明显不符时，把 Avasthas 当"debug 工具"用。
```

## P3 仓库耦合 [参考权重]

```
双宫掌管 = 两个领域捆绑
```

## P5 路况(落宫性质) [重要权重]

行星落在什么类型的宫位：
```

6宫 = 50%效率（竞争/疾病/服务环境）

VRY（Viparita Raja Yoga）= 逆境王格

    Sarala Yoga (8宫主入6/8/12) = 通过危机/继承获得地位
    Vimala Yoga (12宫主入6/8/12) = 低支出或在别人亏损的行业获利
```

**VRY孤立性判定（必须检查，不可跳过）**：
```
VRY不是只看宫主落宫就够了。必须检查该星是否被"污染"：

孤立性条件（决定VRY是否真的生效）：
  与吉宫主(1/5/9/10宫主)同宫且距离>10° → 弱连接
    VRY有效，但附注"福祸并存"
  与吉宫主同宫且距离<5° → 强融合
    VRY被稀释，转为"吉凶混杂"（按P3.2合相处理）
  与吉宫主精确合相<1° → VRY失效
    完全按该吉宫主的资源场分析

强度条件：
  1宫主(Lagna Lord)必须Shadbala>=100%，VRY才可判定为"逆境崛起"
  如果1宫主弱(Shadbala<100%)，VRY降级为"勉强求生"而非"逆境王者"

人话翻译：
  完整VRY = "你有在烂环境里翻盘的天赋"
  被稀释VRY = "你有一些韧性，但不是逆境型选手"
  VRY失效 = "不具备逆境翻盘特质"
```

### P6.2 路况适配度（SAV+BAV交叉判定，必须执行）
```
高SAV + 高BAV = "瞬间超车"（环境好+个人强 → 最佳发挥）
低SAV + 高BAV = "乱世出英雄"（环境差+个人强 → 废土逆袭）
高SAV + 低BAV = "限速封路"（环境好+个人弱 → 浪费机会）
低SAV + 低BAV = "结构性毁灭"（环境差+个人弱 → 无法运作）

BAV阈值（v2 校正，对齐古典 Parashari 标准）：
  BAV 7-8 = 极高（满分附近，完美内驱）
  BAV 5-6 = 偏高（高效内驱，主流强信号）
  BAV 4   = 平均（标准内驱，可正常运作）
  BAV 2-3 = 偏低（内驱不足，需要外部支持）
  BAV 0-1 = 极低（失能，事务难以推进）

⚠️ v1 旧阈值（"5-8=高 BAV"）是错误的——
   它把"偏高+极高"合并为"高"，遗失了重要的力度差异。
   一颗星的 BAV 是 5 还是 7，对应"中等内驱"与"满分内驱"，
   实战表现差异极大，不能合并。
   本次校正与 KN Rao 教学体系一致。
```
