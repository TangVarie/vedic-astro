# 宫位诊断框架

> 用途：core在Step 3宫位诊断时参考
> 每个宫位执行：管理者审计 + 租客审计 + 硬件带宽 + 分盘交叉（如有）

---

## 诊断三维度

### 1. 管理者（宫头星/宫主）
```
查看宫主的：
  P1身份 → 它是什么角色？
  去向(落宫) → 它去了哪个宫位？把这个宫的能量带到了哪里？
  P7尊贵度 → 它健不健康？
  P9 Shadbala → 它有没有力量执行？
```

### 2. 租客（宫内行星）
```
查看宫内星的：
  自带属性 → 吉星进驻=加持，凶星进驻=干扰
  P10相位 → 有没有其他星注视这个宫位？
  合相情况 → 多颗星同宫=事务繁忙
```

### 3. 硬件带宽（SAV）
```
SAV > 32 = 自动化运转
SAV 26-32 = 标准配置
SAV 20-25 = 高阻力
SAV < 20 = 漏雨（需要大量个人努力）
```

## 宫位评级

```
创始人模式：宫主强+宫内有吉星+SAV高 → 自主掌控
经理人模式：宫主中等+SAV中等 → 正常运作
吉祥物模式：宫主弱+有吉星相位 → 有贵人帮但自己弱
飘萍模式：宫主弱+无吉星+SAV低 → 该领域缺乏根基
```

## 多阶段事件推导

大事件不是单一时间点，而是分阶段展开，每个阶段由不同宫主的Dasha触发：

| 事件 | 阶段1 (启动) | 阶段2 (落地) | 阶段3 (收益) |
|------|-------------|-------------|-------------|
| 结婚 | L7 (关系确立) | UL 主 / Jupiter 过 UL / Sun-L7 同步 (法律确认) | L11 (社会公开) |
| 换工作 | L10 (机会/决定) | L6 (入职/竞争) | L11 (收入到位) |
| 买房 | L4 (动念/看房) | L2 (出钱/签约) | L4+D4 (入住) |
| 出国 | L9/L12 (决定) | L12 (离开) | L4 (安家) |

### 婚姻时间推导（三阶段模型，v3 修订）

现代婚姻是过程不是瞬间，Dasha 映射三个阶段：

```
阶段1 — 关系确立/承诺：L7 宫主的 MD/AD
阶段2 — 法律确认(领证)：v3 修订
  → 主指标：UL (Upapada Lagna) 主的 MD/AD，或 Jupiter 行运到 UL
  → 辅助指标：Sun（官方/政府文件指示星）与 L7 / UL 的 Dasha 同步
  → 弃用：v2 曾用 L9 作为法律确认指标，但 L9 主管 dharma/远方/导师/父亲，
         "领证"挂到 L9 的逻辑链不强（L9 与"政府/契约"无直接关联），
         临床命中率偏低。改用 UL（专属婚姻实现点）+ Sun（官方文件）更准。
阶段3 — 社会公开(婚礼/喜宴)：11 宫相关行星的 MD/AD（社交圈见证）

⚠️ Venus 不是婚姻首选触发器，除非 Venus 恰好是 L7 或 UL 主
⚠️ 三个阶段可能在同一个 Antardasha 内，也可能跨 2-3 个
⚠️ 输出格式："关系确立窗口在 [L7 期间]，正式落地在 [UL 主 / Jupiter 过 UL 期间]"
   不要说 "YYYY 年结婚" 这种单点预测

⚠️ UL 计算复习（reader 已提取，如缺失需补算）：
  UL / Upapada Lagna = 第 12 宫的 Arudha Pada（A12 / Vyaya Pada）
  ⚠️ 注意：UL 是从第 12 宫主推出来的，不是从第 7 宫主。把 L7 公式套到 UL 上是常见错误，
     那样算出来的是 A7 / Darapada（配偶的 Arudha），不是 UL。

  标准计算步骤：
  1. 找出第 12 宫（从 Lagna 起的第 12 宫）
  2. 找出第 12 宫主所在星座
  3. 从第 12 宫数到第 12 宫主所在星座，得到距离 N（包含起点和终点）
  4. 从第 12 宫主所在星座再向前数 N 个星座，得到的星座 = 初步 UL
  5. Arudha 例外规则：若初步 UL 落在第 12 宫本身或第 12 宫的对宫（即从 Lagna 看的
     第 12 宫 / 第 6 宫），则从该初步点再数第 10 宫作为最终 UL
  6. UL 主 = UL 所在星座的统治行星
  7. 若与 JHora / Parashara's Light 输出不一致，以软件结果为准（不同流派对 Arudha
     例外的处理略有差异，必须记录所用流派）

  ⚠️ 区分：
    - UL = A12 = 婚姻 / 法律落地点 (Upapada)
    - A7 = Darapada = 配偶在世界中的呈现 (与 UL 不同，常被混淆)
  
  UL 是 Jaimini 体系专属的婚姻实现点，对法律性的婚姻成立最敏感
```

#### 9 宫的实际角色（v3 澄清）

## Dasha事件推导硬约束（必须严格执行）

### 大运正负面判断规则

大运星的正负面不是"感觉"，是有明确判定条件的：

#### 正面条件（满足≥2条→判定为正面）
```
1. P1身份=吉角色（Core-Driver / Yogakaraka / Faithful）
2. P7尊贵度≥友方（Friend/Own/Exalted）
3. P9 Shadbala≥120%
4. 落在吉路（1/4/5/7/9/10宫）
5. 管的宫位SAV≥28
6. Vargottama或Dig Bala加成
```

#### 负面条件（满足任一即标注风险）
```
1. P1身份=凶角色（Trika-Lord 管 8/12 或 Growth-Hacker 管 6）
2. P2受损（燃烧/落陷/败相）
3. 落在凶路（6/8/12宫）且无VRY补救
4. P9 Shadbala<100%
5. 管的宫位SAV<25
6. 与凶星紧密合相（<5°）
```

#### 判定逻辑
```
正面≥2 且 负面=0 → 正面期（该领域顺利/上升）
正面≥2 且 负面≥1 → 混合期（发展中伴随压力）
正面<2 且 负面≥1 → 困难期（该领域面临挑战）
正面=0 且 负面≥2 → 危机期（需谨慎/有风险）
```

### 小运叠加规则
```
大运定基调，小运定具体事件。
小运星管哪个宫→那个领域在该时段被具体激活。

大运正 + 小运正 = 该领域顺利
大运正 + 小运负 = 好事多磨
大运负 + 小运正 = 困境中有转机
大运负 + 小运负 = 该领域面临危机（必须明确警示）
```

### ⚠️ 禁止的推导错误

```
1. 大运星管8宫+落陷 → 禁止说"深度转化带来成长"
   → 必须说"这段时期有突变/危机的风险"
   
2. 大运星管6宫+Shadbala弱 → 禁止说"在竞争中锤炼自己"
   → 必须说"可能面临健康或竞争方面的压力"
   
3. Venus大运+Venus入庙 → 不能因为知道用户那段时间不好就弱化
   → 必须说"这段时期在Venus管的领域有正面能量"
   
4. 凶宫主大运 → 不能美化成"成长的功课"
   → 凶就是凶，可以说"代价"但不能说"礼物"
   
5. 吉宫主大运 → 不能因为用户描述了困难就判负面
   → 吉星大运的正面能量是客观存在的，困难可能来自小运或流年
```

## Badhaka（障碍星）审计

宫位诊断完成后，检查Badhaka宫和Badhakesh（障碍星）：

```
Lagna类型 → Badhaka宫：
  活动星座(Aries/Cancer/Libra/Capricorn) → 11宫
  固定星座(Taurus/Leo/Scorpio/Aquarius) → 9宫
  变动星座(Gemini/Virgo/Sagittarius/Pisces) → 7宫

Badhakesh = Badhaka宫的宫主
  → Badhakesh落入的宫位 = 容易遭遇"莫名障碍"的领域
  → Badhakesh与特定宫位形成关联(合相/相位) → "好事多磨"

在宫位诊断报告中，标注Badhakesh影响的宫位事务。
不过度渲染，仅作为"阻力来源标记"使用。
```
