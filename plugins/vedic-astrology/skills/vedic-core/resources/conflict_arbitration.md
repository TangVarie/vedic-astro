# 多体系冲突裁决矩阵 v0.1（框架版）

> ⚠️ v0.1 框架，非已验证规则。
> 用于：core / career / love / soul / timing 在多体系给出不一致信号时的统一处理。
> 不用于：替代体系本身的解读，也不做"加权打分收敛到单一结论"。
>
> ⚠️ 为什么是框架而不是评分系统：
>   真正的"加权打分"（如 Vim +3 / Chara -2 → 净分判定）需要大量临床样本回测
>   验证权重才有意义。在那之前给出具体分数 = 伪精确，与本系统"不写未验证
>   命中率"原则冲突。本文件给的是**定性三档判定 + 输出措辞模板**，
>   足够避免常见误用（强行收敛 / 单方面信号当铁律），又不假装精确。
>
> ⚠️ 后续路径：积累 50+ 真实盘 × 实际事件的对照样本后，可在此基础上加
>   流派权重和置信度系数，升级到 v1.0 量化版。

## 一、信号层级框架（按时间尺度从慢到快）

```
Layer 1 · 结构层（一生气候）
  数据：D1 + D9 + D10 + Lagna + Functional 矩阵
  时间尺度：终生
  能否单独得结论：能，但只到"潜力 / 倾向 / 上限"层级
  典型问题："这人一生大致什么样"

Layer 2 · 时间层（何时激活）
  数据：Vimshottari MD/AD + Chara MD + Yogini MD
  时间尺度：5-20 年单位
  能否单独得结论：不能，必须叠加 Layer 1 决定"激活什么主题"
  典型问题："这段大运重点是什么"

Layer 3 · 触发层（具体月份/年份）
  数据：Gochara（行运）+ Varshaphala 年盘
  时间尺度：月-年单位
  能否单独得结论：不能，必须叠加 Layer 1 + Layer 2
  典型问题："今年具体什么时候 / 这个月会不会"
```

**核心规则**：上层决定下层的"是否兑现"，下层不能反推上层。
- Layer 1 弱 + Layer 3 强 → 短期波动，不会成为人生主线事件
- Layer 1 强 + Layer 3 弱 → 主题真实，等触发窗口
- Layer 3 强 ≠ Layer 1 强（不能把"今年 Transit 强"当"一生命强"）

## 二、九类典型冲突裁决模板

### 冲突 1：Vimshottari 支持 vs Chara 不支持（同层冲突）

```
两个时间体系（都是 Layer 2）给相反信号。

裁决依据：按主题领域不同，两套体系的临床权重不同。
  ⚠️ 这里的"权重不同"是 KN Rao 体系的临床经验，
     不是实证回测，措辞需保留余地。

  婚姻 / 感情 / 配偶事件
    → Chara Dasha + Jaimini Karaka 体系临床更敏感（Jaimini 专长）
    → 仅 Vim 支持但 Chara 不支持 → 弱结论
    
  事业 / 成就 / 公开身份
    → Vim + Chara 双确认才出硬结论
    → 任一不支持 → 描述为"信号尚未形成共振"
    
  健康 / 重大转折 / 危机
    → 需要 Vim + Chara + Yogini 三体系叠加（Yogini 短周期更敏感）
    → 仅 Vim 支持 → 几乎不出结论
    
  灵性 / 内在转化 / 学习
    → Vim 主线即可，Chara 用作辅助

通用输出模板：
  "Vimshottari 给出 [主题] 的能量窗口，但 Chara Dasha 没有共振，
   可能停留在心理层面或自我探索阶段，不一定爆发为结构性事件。"

反例（这个模板不适用的情况）：
  - 用户软件采用 Chara Dasha 的其他流派（Savya/Apasavya/SJC）
    → 流派差异本身可能造成"虚假冲突"，先核对流派再裁决
```

### 冲突 2：D10 强（职业上升）vs D1 财务宫弱（跨分盘冲突）

```
两个分盘描述同一时间段的不同维度。

裁决依据：D10 与 D1 财务宫描述的不是同一件事。
  D10 = 事业本体（职位、声望、地位、责任）
  D1 第 2 宫 = 个人金钱、积蓄、家庭财
  D1 第 11 宫 = 收入流、增量收益
  D10 强 + D1 财务弱 = 事业上升 ≠ 自动财务回报

裁决：双方都成立，但描述的是分层现实。

通用输出模板：
  "D10 显示事业层上升（[具体格局：升职/曝光/责任加重]），
   但 D1 第 2 / 11 宫提示金钱回报有限。
   常见模式：[学术晋升 / 公益服务 / 创意成就 / 责任增加但薪资滞后]。
   不矛盾，是分层——事业荣耀和财务兑现走的是不同管道。"

反例：
  - 用户具体职业本就高薪 → D1 财务弱可能指别的（如配偶财务、家产）
    → 不要机械套"事业 ≠ 金钱"，结合用户实际情境
  - D10 强是因为有 Raja Yoga，且 Raja Yoga 涉及的星也参与了财富 Yoga
    → 不算冲突，是同一格局的多面表达
```

### 冲突 3：D9 强 vs D1 7 宫弱（跨分盘 + 时间分层冲突）

```
D9 强 = 婚姻 / 灵魂层伴侣关系状态好
D1 第 7 宫弱 = 关系启动 / 表象关系层有压力

裁决依据：D9 是 D1 第 7 宫的"下一层映射"，描述婚后状态而非婚前过程。

裁决：两者描述的是关系的不同阶段。

通用输出模板：
  "关系启动可能困难或波折（D1 第 7 宫弱），
   但一旦进入正式婚姻状态会改善（D9 强）。
   典型模式：
     - 晚婚 / 多次相亲 / 关系前期波折 / 婚后稳定
     - 第一段关系不成 / 第二段才入婚姻
     - 婚前长期同居调整 / 入婚后突然契合"

反例：
  - DK / UL 同时弱 → 不能简单用"婚后改善"，要重新评估婚姻是否实际发生
  - L7 与 D9 Lagna 主之间有强烈相位冲突 → 描述"婚姻有但伴侣本身有挑战"

### 冲突 4：Transit 强 vs Dasha 不激活（跨时间尺度冲突）

```
Layer 3（Transit / Gochara）强 + Layer 2（Vim/Chara Dasha）不激活

   短期内可见 [情绪波动 / 机会窗口 / 关注转移]。
   但当前 Dasha（[当前 MD/AD]）主题不在此领域，
   所以这段能量更多是'天气变化'，不太可能转为'气候转折'。
   可以利用这段窗口做 [短期行动建议]，但不必为之做重大决策。"

反例：

### 冲突 5：Varshaphala 强 vs 本命承载弱（年盘 vs 本命冲突）

```
Varshaphala（Layer 3）强 + 本命（Layer 1）相应主题弱

裁决依据：年盘可以触发本命范围内的"高水位"，但不能突破本命上限。
  本命 = 一生气候上限
  年盘 = 本年具体天气，可达本命允许的上限附近，但不能超越

裁决：年盘信号真实，但放大效应受本命限制。

通用输出模板：
  "Varshaphala 显示本年在 [主题] 方面有强机会窗口
   ([Year Lord 强 / Muntha 在 X 宫 / Saham 激活])。
   但本盘相应配置 ([相关宫主 / Karaka / 分盘]) 承载力有限。
   解读：本年可以在本命允许的天花板附近充分发挥，
        但不要期待本年的高水位成为新常态。
   可行动作：抓住本年窗口完成 [具体可达的目标]，
            不要把它当作'人生主题的突破点'去过度投入。"

反例：
  - 本命弱但本年 Year Lord = 本命强星 → 实际上是激活了本命某条次线
    → 不算冲突，是本命隐藏潜能的暴露
  - 本年配置触发完整 Raja Yoga 的成员 → 可能产生"本年高于一生平均"的真实跃迁
    → 不机械套此模板
```

### 冲突 6：L1 强 vs L4 弱（同分盘宫位冲突）

```
本盘 Lagna 主（L1）强 + 第 4 宫主（L4）弱
= 自我能量充足，但家庭 / 根基 / 内在情绪稳定性薄弱。

裁决依据：L1 和 L4 在 D1 内描述不同人生轴线：
  L1 = "向外的我"（行动力、自我认同、健康外显）
  L4 = "向内的我"（家庭、母亲、内心安全感、不动产、最后的归处）
  两者各自独立成立，不冲突，是命盘的结构性特征。

裁决：双方都成立，是性格 / 命运的双轴呈现，不是矛盾。

通用输出模板：
  "本盘呈现典型的'外强内不稳'结构（L1 强 + L4 弱）：
   你在社会层面 / 行动层面有较强能量，
   但内在情绪安全感、家庭根基、与母亲 / 童年家的连接相对薄弱。
   
   典型表现：
     - 外人看你能干 / 有冲劲 / 自信，内在却常有不安或孤独感
     - 容易离开出生地 / 跟原生家庭关系疏远 / 不动产相关事项多波折
     - '外在成就感' ≠ '内心安宁'，两者需要分别建设
     - 适合通过 [心理咨询 / 家庭关系修复 / 稳定居所 / 静观练习]
       主动补 L4 主题"

反例：
  - L4 主弱仅因落 6/8/12 但合相 L1 主 → '外强内不稳'被外在能量
    部分覆盖，描述为"靠行动消化内在不安"而非简单"内在弱"
  - Cancer Lagna 时 Moon 既是 L1 又是自然 4 宫支配者
    → 不能简单按"L1 vs L4"二分处理，需结合 Moon 状态单独判
  - 类比扩展：L1 强 + L7 弱（关系层）、L1 强 + L10 弱（事业层）、
    L1 强 + L5 弱（创造层）适用同一框架，替换具体宫位主题即可。
```

### 冲突 7：BAV 高分宫 vs 该宫宫主弱（宫位资源 vs 主星冲突）

```
某宫 Ashtakavarga 分数高（BAV / SAV ≥ 30），
但该宫的宫主在本盘弱化（落陷 / 燃烧 / 受克 / 入 Dustana / 战败）。

裁决依据：Ashtakavarga 是"宫位资源储备"，宫主是"宫位执行人"。
  - 资源好但执行人差 → 资源在，但调用不上
  - 资源差但执行人强 → 执行人弥补资源不足
  这是 KN Rao 反复强调的"双指标交叉验证"。
  ⚠️ 这条裁决基于 KN Rao 的体系性判断，
     不要倒过来用"BAV 高 → 一定顺"或"宫主弱 → 一定不顺"单边推断。

裁决：宫位主题的兑现度受双指标共同约束，单看任一指标都误判。

通用输出模板：
  "本盘 [X 宫] BAV / SAV [数值] 高，说明这个宫位的'资源储备'好；
   但 [X 宫] 主 [行星] 在本盘 [具体弱化状态]，
   说明负责调用这些资源的'执行人'本身不够给力。
   
   典型表现：
     - 资源齐备但启动慢 / 关键时刻使不上劲 / 总差一步
     - 别人看着你这方面条件好，但你自己知道实际操作起来吃力
     - 该主题的事件在生命中真的会发生，但耗时长 / 波折多 / 回报折扣
   
   突破口：
     - 等 [X 宫主] 进入主大运 / 重要 Antardasha → 执行人被激活，
       届时积累的资源会集中兑现
     - 中间期借力外部资源 / 合作伙伴 / 顾问，弥补自身执行力短板
     - 不要因为'BAV 高'盲目自信，也不要因为'宫主弱'放弃"

反例：
  - 宫主在 Kendra（即使弱化）→ 弱化程度被 Kendra 位置部分抵消
    → 不一定"使不上劲"，可能只是"晚发动"
  - 宫主入 Dustana（6/8/12）+ BAV 高 → 这是 Vipreet Raja Yoga 类的
    "反向加持"结构，资源会通过非常规途径变现，不适用本模板
  - 多颗行星共同管该宫（如 Aquarius Lagna 的 11 宫只有 Saturn 一个主，
    但 Capricorn 11 宫的主有 Saturn + Rahu 副主）→ 双主同时弱才算冲突
```

### 冲突 8：自然吉星 vs Functional Malefic（自然属性 vs 功能属性冲突）

```
某颗自然吉星（Jupiter / Venus / Mercury / 强 Moon）
在本盘是 Functional Malefic（管的宫位组合在本 Lagna 下拖累命主）。

典型场景（参考 vedic-core/resources/p1_p12.md 矩阵）：
  - Aries Lagna 的 Venus（L2 + L7 双 Maraka）
  - Taurus Lagna 的 Jupiter（L8 + L11 双 dusthana/upachaya）
  - Cancer Lagna 的 Venus（L4 + L11 Kendradhipati）
  - Cancer Lagna 的 Mercury（L3 + L12）
  - Leo Lagna 的 Mercury（L2 + L11）
  - Libra Lagna 的 Jupiter（L3 + L6 严重 FM）
  - Sagittarius Lagna 的 Venus（L6 + L11 严重 FM）
  - Capricorn Lagna 的 Jupiter（L3 + L12）

裁决依据：自然属性 ≠ 功能属性（p1_p12.md P1.3 纹理检查的核心）。
  自然吉星 = 该星本身能量"温和 / 滋养 / 智慧"
  Functional Malefic = 该星在本 Lagna 下管的宫位组合"拖累命主"
  
  这是 AI 最容易犯的错误：看到 Jupiter / Venus 就说"好"，
  忽视它在本 Lagna 下的 functional 角色。

裁决：该星的大运 / 行运是"欺骗性窗口"——表面温和实际拖累。

通用输出模板：
  "[行星名] 是自然吉星，但在本盘 [Lagna 名] 下管 [宫位组合]，
   功能上是 Functional Malefic。
   
   实战意义：
     - [行星] 的大运 / Antardasha 不是'吉运'，而是'温柔陷阱'
     - 这段时期看起来风平浪静 / 机会多 / 人际好，
       但实际进展会停滞 / 资源被消耗 / 表面安稳但本质损耗
     - 容易被'感觉良好'误导而做错决策
   
   关键应对：
     - 不要在 [行星] 大运做重大投入 / 长期承诺 / 大笔投资
     - 利用这段窗口休整、做不涉及关键决策的辅助性建设
     - 等下一颗 Functional Benefic 的 Dasha 再启动主线"

反例：
  - 该 FM 行星同时是 Yogakaraka（如 Taurus Saturn 既是自然凶星
    又是 Yogakaraka）→ 不适用本模板，按 Yogakaraka 处理
  - 该 FM 行星在自己的入庙 / 旺宫 → 状态可部分抵消功能拖累
    → 描述为"温和拖累但有底线"，不要全盘否定
  - 用户已在该 Dasha 中且已做出重大决定 → 不要事后否定，
    转为分析"决策已成，如何在此期间防御 / 微调"
  - 反向情况：自然凶星（Saturn / Mars）+ Functional Benefic → 
    "苦但有产出"，与本冲突镜像，但措辞方向相反
```

### 冲突 9：Karakamsa 主题 vs D1 Lagna 主题（灵魂层 vs 现世层冲突）

```
Karakamsa Lagna（AK 在 D9 的星座）显示的灵魂主题
与 D1 Lagna 显示的现世身份方向不一致。

裁决依据：两者描述不同层面（参考 vedic-soul SJC 体系）：
  D1 Lagna = 现世身份 / 社会角色 / 此生的"职业头衔"
  Karakamsa Lagna = 灵魂选择的功课主题 / 此生的"内在使命"
  
  二者一致 → 内外合一，事业即天命
  二者不一致 → 双轨结构，需要分别建设

裁决：不是矛盾，是双轨。强行让灵魂方向服从现世（或反过来）都会失衡。

通用输出模板：
  "D1 Lagna ([sign]) 显示的现世身份偏向 [现世方向特质]，
   但 Karakamsa Lagna ([AK 在 D9 的 sign]) 显示的灵魂主题是 [灵魂方向特质]。
   
   两者不一致，但这不是矛盾，是双轨结构。常见呈现：
     - 主业是 [现世方向]，业余 / 内在追求 [灵魂方向]
     - 前半生走 [现世方向]，中年后转向 [灵魂方向]
       （典型 Mercury MD 或 Saturn → Mercury 过渡期）
     - 表面身份 [现世方向]，但真正让你感到'活着'的活动属于 [灵魂方向]
   
   不健康的处理：
     - 完全压抑灵魂方向追求世俗成功 → 中年危机 / 抑郁 / 意义感丧失
     - 完全放弃现世身份只走灵魂方向 → 经济压力 / 关系压力反噬
   
   健康的处理：
     - 现世方向做'基础设施'（养活自己 + 给灵魂方向腾出空间）
     - 灵魂方向做'真实表达'（不必为变现，但不可彻底放弃）
     - 在 D10 强的 Dasha 期推现世，在 AK 主 Dasha 期推灵魂"

反例：
  - D1 Lagna 主和 AK 是同一颗行星 → 两轨实际同源，不算冲突
  - Karakamsa 主和 D1 Lagna 主有强相位（合相 / 三合 / 互照）
    → 两轨已有桥梁，描述为"灵魂方向通过现世身份表达"而非张力
  - 用户已明确选择某一轨为主线（已 50+ 岁且方向稳固）
    → 不强加"双轨"叙述，转为分析当前主线下另一轨如何辅助
  - Karakamsa Lagna 落 12 宫起算的特殊位置（Karakamsa 在 D1 第 12 宫）
    → 这本身就是"出世 / 隐居"主题，与多数 D1 Lagna 都会显出张力，
       属于体系特征而非个案冲突
```

## 三、通用三档裁决原则（适用于所有冲突类型）

```
档位 1 · 双确认 → 强结论
  两套独立体系（不同 Layer / 不同分盘 / 不同流派）同方向支持
  → 输出措辞：明确、具体、可时间锁定
  → 例："2026 年下半年很可能 [事件]"
  → 置信度标签：高

档位 2 · 单方面强 → 弱结论
  仅一个体系强支持，另一体系中性或弱反对
  → 输出措辞：保留余地，"可能 / 部分 / 倾向于"
  → 例："2026 年下半年有 [主题] 的能量倾向"
  → 置信度标签：中

档位 3 · 真矛盾 → 描述张力，不强行收敛
  多个体系给相反信号且各自有强支撑
  → 输出措辞：描述张力本身
  → 例："本盘在此主题上呈现两股相反力量——[体系 A 给出方向 X]，
       [体系 B 给出方向 Y]。这通常表现为 [行为层面的实际呈现]。"
  → 置信度标签：描述性，非预测性
```

⚠️ 档位判定不是机械的"几个体系支持"投票，而是看：
  1. 体系本身在该领域是否权威（如婚姻看 Chara/Jaimini 更准）
  2. 体系内信号本身的强度（弱支持 ≠ 强支持）
  3. 体系之间是否真的独立（D9 和 D1 第 7 宫不算完全独立）

## 五、不做的事（与本系统一贯原则一致）

```
❌ 不给具体分数（如"Vim +3 / Chara -2 = 净分 1"）
   - 没有样本回测，分数 = 伪精确
   
❌ 不机械投票收敛（如"3 个体系支持 vs 2 个反对 → 走多数")
   - 不同体系权威不同，投票忽视权威差异
   
❌ 不强行得出单一结论
   - 真矛盾时描述张力本身比硬选一方诚实
   
❌ 不把"信号弱"等同于"信号反向"
   - 中性 ≠ 反对
```
