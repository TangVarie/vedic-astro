# 手动时间扫描 Fallback 方案

> 用途：当 time_scan.py 脚本不可用时（ephem 未安装 / 环境问题 / 脚本报错），
>      使用手动估算方法完成时间扫描
> 精度：±8分钟（低于脚本的±5分钟，但足以判断Lagna是否需要换座）

---

## 触发条件

```
time_scan.py 执行失败 → 错误类型判断：
  ImportError (ephem) → pip install ephem>=4.1 --break-system-packages
                        或 pip install -r vedic-rectifier/requirements.txt
  ⚠️ 当前脚本依赖 PyEphem（包名 ephem），不是 pyswisseph。
     若未来升级为 Swiss Ephemeris 版本，再另建 time_scan_swe.py 或
     修改 requirements，目前不要安装 pyswisseph。
  其他 ImportError → 尝试安装缺失包
  安装后仍失败    → 启用手动 fallback
  脚本文件不存在  → 启用手动 fallback
```

---

## 手动Lagna换座时间估算

### 核心公式（v3 修订）

```
Lagna 在黄道上的移动速度 ≈ 360° / 24 小时 = 15°/小时 = 0.25°/分钟

一个星座跨度 = 30°
Lagna 穿过一个星座的"赤道平均时间" ≈ 30° / 15°/小时 = 2 小时

⚠️⚠️⚠️ v3 重要修正：以上是"赤道近似"，不能直接套用到中高纬度地区！

实际 Lagna 速度取决于：
  1. 出生纬度
  2. 出生季节（黄赤交角的相对位置）
  3. 具体星座（不同星座上升角度不同）

不同星座的"实际上升时间"差异（在 30°N 纬度，分钟）：
  Aries:       97-115 分钟  Libra:       115-135 分钟
  Taurus:      105-125 分钟  Scorpio:     130-150 分钟
  Gemini:      125-145 分钟  Sagittarius: 130-145 分钟
  Cancer:      135-150 分钟  Capricorn:   105-125 分钟
  Leo:         135-150 分钟  Aquarius:    100-115 分钟
  Virgo:       130-145 分钟  Pisces:      97-115 分钟

到了 40°N 以上（北京、纽约、北京以北全部）：
  长升星座（Cancer/Leo/Virgo/Libra/Scorpio/Sagittarius）= 上升时间显著 > 2 小时
    可达 150-200 分钟
  短升星座（Capricorn/Aquarius/Pisces/Aries/Taurus/Gemini）= 上升时间显著 < 2 小时
    可短至 60-100 分钟

到了 60°N 以上（高纬度，如挪威、芬兰、阿拉斯加）：
  某些星座极速上升（不到 30 分钟）
  某些星座极慢上升（超过 4 小时）
  本 fallback 公式精度严重失效，必须依赖天文计算
```

### 纬度修正系数（v3 新增）

```
快速估算的"纬度修正"（在 2 小时基础上修正）：

| 纬度范围 | 长升星座修正 | 短升星座修正 | 精度损失 |
|---------|-------------|------------|---------|
| 0°-25° (热带) | +0~5 分钟/星座 | -0~5 分钟/星座 | ±3 分钟 |
| 25°-40° (副热带) | +10~20 分钟 | -10~20 分钟 | ±5 分钟 |
| 40°-55° (温带) | +20~40 分钟 | -20~40 分钟 | ±10 分钟 |
| 55°-65° (亚北极) | +40~80 分钟 | -40~80 分钟 | ±20 分钟 |
| > 65° (极地附近) | 不适用 | 不适用 | 失效 |

中国大陆主要城市纬度参考：
  广州/深圳:  ~23°N  (热带边缘，公式精度 ±3-5 分钟)
  上海:       ~31°N  (副热带，公式精度 ±5-7 分钟)
  北京/天津:  ~40°N  (温带，公式精度 ±10 分钟)
  哈尔滨/长春: ~45°N (温带北部，公式精度 ±10-15 分钟)
  漠河:      ~53°N  (亚北极边缘，公式精度 ±15-20 分钟)

修正规则：
  Step 1: 用基础公式计算理论换座时间（30°-X°）× 4 分钟
  Step 2: 看出生城市纬度，从上表查精度损失范围
  Step 3: 在输出中标注：
    "基于赤道近似估算，[城市] 纬度下精度约 ±[N] 分钟"
  Step 4: 如纬度 > 40° 且星座是长升类，建议把换座窗口扩大 20-40 分钟双向扫描
```

### Lagna 换座时间点估算（v3 修订）

```
已知：当前Lagna = [星座], 度数 = [X°]，出生城市纬度 = [Lat°N]

Step 1: 基础公式（赤道近似）
  到下一星座的理论时间 = (30° - X°) × 4 分钟
  到上一星座的理论时间 = X° × 4 分钟

Step 2: 纬度修正（v3 新增）
  查纬度修正表 → 修正系数 ±N 分钟
  
  长升星座当前段 + 高纬度 → 实际换座时间 > 理论时间
  短升星座当前段 + 高纬度 → 实际换座时间 < 理论时间

Step 3: 扫描范围扩大（v3 新增）
  如纬度 > 40° → 把基础扫描范围 ±15/30/60 分钟 扩大 50%
    （即 ±22/45/90 分钟），覆盖不确定区间

示例（v3 含纬度修正）：
  Lagna = Cancer 28°15'（Cancer 是长升星座）
  出生地：北京（40°N）
  
  基础公式：(30-28.25) × 4 = 7 分钟后换 Leo
  纬度修正：Cancer 在 40°N 升时间 +20~30 分钟/全星座，
            按比例缩放：1.75° 剩余对应 ≈ 10-12 分钟（理论 7 分钟）
  
  → 输出建议："Lagna 距 Leo 换座约 7-12 分钟（赤道理论 7 分钟，
                40°N 纬度修正后实际 10-12 分钟），扫描范围扩大到 ±22 分钟"
```

### 手动扫描执行步骤

```
Step 1：计算Lagna换座的两个时间分界点（前/后）
Step 2：确定哪个分界点在扫描范围内
  扫描范围 = ±[15/30/60]分钟（由Step 1确定）
Step 3：如果分界点在扫描范围内
  → 需要对比两个Lagna的事件匹配度
  → 构造两个宫主表（当前Lagna vs 替代Lagna）
  → 按3c的双盘对比评分表执行
Step 4：如果分界点不在扫描范围内
  → Lagna不变，只需在当前Lagna内微调
  → 检查D9换座点（约每13分钟一次）做D9级微调
```

### D9换座时间点估算

```
D9 Navamsha 每等分 = 3°20' = 200' = 10/3 度
   ⚠️ 用 3.333 近似会在边界附近累积漂移 (3.333 × 9 = 29.997，不是 30)
Lagna 移动 10/3 度的时间 ≈ (10/3) × 4 = 40/3 ≈ 13.33 分钟

从当前 Lagna 度数计算当前 Navamsha 位置（精确版）：
  绝对经度（分钟） = 度 × 60 + 分
  Navamsha 序号 = floor(绝对经度分钟 / 200)
  当前 Navamsha 内已走分钟数 = 绝对经度分钟 mod 200
  到下一 Navamsha 的分钟数（时间） = (200 - 余数分钟) × (4/60) 分钟
  到上一 Navamsha 的分钟数（时间） = 余数分钟 × (4/60) 分钟

近似版（边界外可用）：
  Navamsha 序号 ≈ floor(绝对经度 / (10/3))
  到下一 Navamsha 的时间 ≈ ((10/3) - 度数余数) × 4 分钟
```

### 宫主表快速构造（不同Lagna）

```
如果Lagna变化 → 所有12宫的星座都shift → 宫主表完全改变

快速构造法：
  新Lagna = [星座]
  1宫 = 新Lagna星座 → 宫主 = 该星座的ruling planet
  2宫 = 新Lagna+1 → 宫主 = ...
  ...以此类推12个宫

星座→宫主速查：
  Aries → Mars        Taurus → Venus      Gemini → Mercury
  Cancer → Moon        Leo → Sun           Virgo → Mercury
  Libra → Venus        Scorpio → Mars      Sagittarius → Jupiter
  Capricorn → Saturn   Aquarius → Saturn   Pisces → Jupiter
```

```
     40°-55°N：约 ±15-20 分钟

```
